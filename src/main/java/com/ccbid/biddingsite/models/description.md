# What this folder does:

This folder contains all of the models for the application.

Models are essentially the classes that represent the actual data and users of the application.

This includes auctioneers, account roles, bids, items, and more.

The models are used to create objects that represent the data in the application, and they often include methods for manipulating that data.

- Specific methods and how to name them were generated by Codex with the prompt of "I need to create a Spring method that finds active orders based on users" for certain services and it would generate the methods as needed.
